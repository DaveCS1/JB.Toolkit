\doxysubsection{Detailed Description}
Uses Open\+X\+ML to perform a Word Mail Merge. 

Microsoft Word needs to be installed on the executing machine in order to use this. It uses an Interop A\+PI, which means to perform the mail merge, Word needs to be opened in the background, then the mail merge is run and finally office is closed down.

This is the most reliable in terms of output format, and you can extend to run macros, however this isn\textquotesingle{}t the best means in terms of efficiency and it isn\textquotesingle{}t cross user thread safe. So if you\textquotesingle{}re wanting to use as part of a web service / web site running on I\+IS, you\textquotesingle{}ll need to make sure Office is installed, you employ lock / wait timers and additional configurations are setup, refer to this guides\+:

\href{https://www.ryadel.com/en/office-interop-dcom-config-windows-server-iis-word-excel-access-asp-net-c-sharp/}{\texttt{ https\+://www.\+ryadel.\+com/en/office-\/interop-\/dcom-\/config-\/windows-\/server-\/iis-\/word-\/excel-\/access-\/asp-\/net-\/c-\/sharp/}} \href{http://www.bloing.net/2011/01/how-to-make-iis7-play-nice-with-office-interop/}{\texttt{ http\+://www.\+bloing.\+net/2011/01/how-\/to-\/make-\/iis7-\/play-\/nice-\/with-\/office-\/interop/}}

Don\textquotesingle{}t use if the class will get hammered. Either try the Open\+X\+ML implementation in this toolkit or speak to someone to invest in Aspose software.
