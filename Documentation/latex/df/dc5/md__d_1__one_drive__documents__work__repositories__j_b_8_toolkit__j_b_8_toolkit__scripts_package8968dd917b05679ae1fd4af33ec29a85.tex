The datatable accepts the following callback functions as arguments\+: \begin{DoxyVerb}onAddRow(alteditor, rowdata, success, error)
onEditRow(alteditor, rowdata, success, error)
onDeleteRow(alteditor, rowdata, success, error)
\end{DoxyVerb}


In the most common case, these function should call {\ttfamily \$.ajax} as expected by the webservice. The two functions {\ttfamily success} and {\ttfamily error} should be passed as arguments to {\ttfamily \$.ajax}.

Within the procedures {\ttfamily on\+Add\+Row}, {\ttfamily on\+Edit\+Row}, {\ttfamily on\+Delete\+Row}, you can access datatable object using {\ttfamily alteditor.\+s.\+dt}.

Webservice {\bfseries{must}} return the modified row in J\+S\+ON format, because the {\ttfamily success()} function expects this. Otherwise you have to write your own {\ttfamily success()} callback (e.\+g. refreshing the whole table).\hypertarget{md__d_1__one_drive__documents__work__repositories__j_b_8_toolkit__j_b_8_toolkit__scripts_package8968dd917b05679ae1fd4af33ec29a85_autotoc_md7}{}\doxysubsection{Row key}\label{md__d_1__one_drive__documents__work__repositories__j_b_8_toolkit__j_b_8_toolkit__scripts_package8968dd917b05679ae1fd4af33ec29a85_autotoc_md7}
There is no default key in the table. Inside your callback functions, probably you will need a row key to build U\+RL\textquotesingle{}s, in that case you can get them from the {\ttfamily rowdata} parameter.\hypertarget{md__d_1__one_drive__documents__work__repositories__j_b_8_toolkit__j_b_8_toolkit__scripts_package8968dd917b05679ae1fd4af33ec29a85_autotoc_md8}{}\doxysubsection{Column id}\label{md__d_1__one_drive__documents__work__repositories__j_b_8_toolkit__j_b_8_toolkit__scripts_package8968dd917b05679ae1fd4af33ec29a85_autotoc_md8}
